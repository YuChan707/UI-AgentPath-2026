# OnLooker — Synthetic audience pipeline workflow

End-to-end flow: from real Census data (NY) to synthetic audience groups with
scores, anchored in real evidence (Foundry IQ) and generated by a dockerized
Llama 3B via Dapr.

## 1. Overview

```mermaid
flowchart TD
    subgraph INGEST["data_ingestor (real RAW data)"]
        CSV["nyc_zip_codes.csv<br/>(NY zip codes)"]
        CENSUS["US Census ACS5 API<br/>(CENSUS_DATA_API)"]
        BUILD["build_location_metadata<br/>+ Marshmallow Location.load()"]
        PERSIST_LOC["persist_locations()"]
        LOCJSON[("data_ingestor/data/locations/<br/>&lt;zip&gt;.json + _index.json")]
        CSV --> BUILD
        CENSUS --> BUILD
        BUILD --> PERSIST_LOC --> LOCJSON
    end

    subgraph PROC["data_processor (SYNTHETIC data $$)"]
        LOAD["load_locations()"]
        P1["1. Statistical model<br/>BEHAVIOR_MODEL_PROMPT<br/>-> list[BehaviorFormula]"]
        P2["2. Groups by field/topic<br/>FIELD_GROUPS_PROMPT<br/>-> list[FieldBehaviorGroup]"]
        P3["3. Varied groups (automatic)<br/>build_varied_group_definitions<br/>+ GROUP_PROFILE_PROMPT<br/>-> GroupBehaviorProfile"]
        P4["4. summarize_scores()<br/>consolidated spec + audience_scores"]
        SAVE["save_processor_output / save_audience_specs"]
        OUTJSON[("data_processor/output/<br/>behavior_model / field_groups /<br/>group_profiles / audience_specs")]
        LOAD --> P1 --> P2 --> P3 --> P4 --> SAVE --> OUTJSON
    end

    subgraph LLM["Model access + grounding"]
        FIQ["FoundryIQ.ground()<br/>real evidence (Census) /<br/>remote retrieval"]
        CLIENT["LlamaClient<br/>auto: dapr -> http -> mock"]
        DAPR["Dapr Conversation API<br/>component: llama"]
        LLAMA["dockerized Llama 3B<br/>(OpenAI-compatible)"]
        FIQ --> CLIENT
        CLIENT --> DAPR --> LLAMA
    end

    LOCJSON --> LOAD
    P1 -.uses.-> CLIENT
    P2 -.uses.-> CLIENT
    P3 -.uses.-> CLIENT
    OUTJSON --> UI["OnLooker UI /<br/>backend (reactions to an asset)"]
```

## 2. Validated generation (a PromptSpec)

Each generation step goes through the same cycle, with strict validation against
the schema and degradation to mock if there is no model.

```mermaid
flowchart TD
    START["generate(PromptSpec)"]
    GROUND["FoundryIQ.ground(query, stats)<br/>appends real evidence to the prompt"]
    BUILDP["spec.build() -> {model, system, user}"]
    ISMOCK{"transport == mock<br/>or LLM not available?"}
    CALL["LlamaClient.complete()"]
    JSON["extract_json(response)"]
    VALID{"spec.validate()<br/>(Marshmallow .load)"}
    RETRY{"retry &lt; 2?"}
    MOCK["spec.mock() -> valid fixture"]
    OK["validated entity"]

    START --> GROUND --> BUILDP --> ISMOCK
    ISMOCK -- no --> CALL --> JSON --> VALID
    VALID -- ok --> OK
    VALID -- fails --> RETRY
    RETRY -- yes --> CALL
    RETRY -- no --> MOCK
    ISMOCK -- yes --> MOCK
    MOCK --> VALID2["spec.validate() of the mock"] --> OK
```

## 3. Data model (key entities)

```mermaid
flowchart LR
    LOC["Location<br/>(+ LocationStatistics)"]
    BF["BehaviorFormula<br/>(baseline + FactorModifier[])"]
    FG["FieldBehaviorGroup<br/>(tech/edu/entert/health...)"]
    GP["GroupBehaviorProfile<br/>(behavior_ranges = MetricRange[])"]
    SPEC["audience_spec<br/>(aggregated audience_scores)"]
    RP["ReactionProfile<br/>(scores toward an asset)"]

    LOC --> BF
    LOC --> FG
    LOC --> GP
    BF --> GP
    BF --> FG
    GP --> SPEC
    FG --> SPEC
    GP --> RP
    SPEC --> RP
```

## How to run

```bash
# 1) Real raw data -> persisted by zip_code
python -m data_ingestor.main                 # requires CENSUS_DATA_API in .env

# 2) Synthetic audience (auto: dapr -> http -> mock)
python -m data_processor

# "Llama via Dapr agents" path
dapr run --app-id data-processor \
         --resources-path data_processor/components \
         -- python -m data_processor --transport dapr

# Offline demo (no model)
python -m data_processor --transport mock
```
